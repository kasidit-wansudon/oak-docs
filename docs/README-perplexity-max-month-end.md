# README — วิธีใช้ Perplexity Max ให้คุ้มภายในสิ้นเดือน

เอกสารนี้สรุปแนวทางใช้ Perplexity Max สำหรับงานของ Kasidit ให้คุ้มที่สุดภายในช่วงเวลาที่เหลือของเดือน โดยอิงจากบริบทงานจริงใน Asana, Notion, GitHub, Google Drive, ปฏิทิน, ข้อมูลทางการของ Perplexity, งานวิจัย, และสัญญาณจากโซเชียล

## TL;DR

ใช้ Perplexity Max กับงานมูลค่าสูง 4 ก้อนก่อน:

- ปิด backlog กลุ่ม DingTalk/HR ใน OPPO MEX
- ทำ résumé + cover letter สำหรับ remote job
- จัด GitHub portfolio และ README ของ repo หลัก
- สรุป architecture decision ของ OpenClaw ให้ลงมือทำต่อได้

## ทำไมแผนนี้เหมาะกับคุณ

จากสัญญาณในงานปัจจุบัน งานเร่งของคุณกระจุกอยู่ที่บั๊กฝั่ง DingTalk/HR บนโปรเจกต์ OPPO MEX ([Asana OPPO MEX](https://app.asana.com/1/1203190425977344/project/1212608080690500), [Leave record management function](https://app.asana.com/1/1203190425977344/project/1212608080690500/task/1213484775751595), [Position block from HR](https://app.asana.com/1/1203190425977344/project/1212608080690500/task/1213393885260127)).

## Perplexity Max ให้อะไรเพิ่ม

Perplexity Max รวมทุกอย่างใน Pro และเพิ่มสิทธิ์เข้าถึง advanced models ในระดับสูงกว่า, สิทธิ์ใช้ Create files and apps มากขึ้น, early access ฟีเจอร์ใหม่, และ priority support ([Perplexity Max Help Center](https://www.perplexity.ai/help-center/en/articles/11680686-perplexity-max), [Plan comparison](https://www.perplexity.ai/help-center/en/articles/11187416-which-perplexity-subscription-plan-is-right-for-you)).

บล็อกเปิดตัวระบุว่า Max เหมาะกับคนที่ต้องการ productivity สูง, งานวิเคราะห์ลึก, competitive intelligence, และการสร้าง dashboards, spreadsheets, presentations, และ web apps ([Introducing Perplexity Max](https://www.perplexity.ai/hub/blog/introducing-perplexity-max)).

Changelog ล่าสุดยังเพิ่ม Model Council, memory ที่ดีขึ้น, และ browser agent ที่เลือกโมเดลได้ โดย Max ใช้ตัวเลือก reasoning ที่แรงกว่าในงานซับซ้อน ([What We Shipped - February 6th, 2026](https://www.perplexity.ai/changelog/what-we-shipped---february-6th-2026)).

## วิธีใช้ให้คุ้ม

### 1. เริ่มจาก Research ก่อน

สำหรับงานของคุณ ควรเริ่มด้วย Research หรือ Deep Research ก่อนเกือบทุกครั้ง เพราะงานส่วนใหญ่เป็น debugging, architecture, job research, และ technical comparison ที่ต้องพึ่งข้อมูลใหม่และหลายแหล่ง ([How Credits Work](https://www.perplexity.ai/help-center/en/articles/13838041-how-credits-work-on-perplexity), [Introducing Perplexity Max](https://www.perplexity.ai/hub/blog/introducing-perplexity-max)).

### 2. ใช้ Create files and apps เมื่ออยากได้ของส่งต่อ

เมื่อปลายทางเป็น résumé, README, architecture note, pricing sheet, slide หรือ test plan ให้ใช้ Create files and apps เพราะถูกออกแบบมาสำหรับการสร้าง artifact โดยตรง ([Perplexity Create files and apps](https://www.perplexity.ai/help-center/en/articles/11144811-perplexity-create-files-and-apps)).

### 3. ใช้ Computer เฉพาะงานที่ต้องเข้าเว็บจริงหลายขั้น

Computer ควรเก็บไว้ใช้กับงานที่ต้องเปิดเว็บจริง, เดิน flow, หรืออ่าน dashboard แบบ interactive เช่น ตรวจพฤติกรรมหน้า UI, เช็ก career pages, หรือวิเคราะห์ onboarding ของคู่แข่ง ([How Credits Work](https://www.perplexity.ai/help-center/en/articles/13838041-how-credits-work-on-perplexity), [What We Shipped - February 6th, 2026](https://www.perplexity.ai/changelog/what-we-shipped---february-6th-2026)).

### 4. ใช้ Model Council กับโจทย์ที่เดิมพันสูง

Model Council ควรใช้กับโจทย์ที่ไม่อยากเชื่อ model เดียว เช่น เลือก local model สำหรับ RTX 4060, วาง routing policy ของ OpenClaw, หรือเทียบ stack ของ RAG สำหรับ `ai-hr-analytics` ([What We Shipped - February 6th, 2026](https://www.perplexity.ai/changelog/what-we-shipped---february-6th-2026), [OpenClaw Server Planning & AI Architecture](https://www.notion.so/325c16912d2180f7b862f092d314d0ab), [ai-hr-analytics](https://github.com/kasidit-wansudon/ai-hr-analytics)).

## พร้อมใช้ทันที

### Prompt: DingTalk backlog

```text
Act as a senior Laravel + enterprise workflow debugger.
I have multiple related bugs around DingTalk role sync, leave records, resign flow, and position-based access control.
My goal is to identify the likely shared root causes first, not fix symptoms one by one.
Please do the following:
1) map the possible shared architecture failures
2) list the top 5 root-cause hypotheses with evidence I should verify
3) generate a regression checklist
4) generate test cases in table format
5) tell me what logs and DB queries I should inspect first
Context: PHP/Laravel backend, HR/leave management, DingTalk integration, enterprise role hierarchy.
Output in Thai, but keep code/comments/examples in English when relevant.
```

### Prompt: AU resume

```text
Create an Australia-format senior developer resume for me.
Position me as a Full-stack Developer / Technical PM with strong Laravel, Vue.js, API integration, AWS, AI-assisted development, and enterprise HR/WMS experience.
Use measurable impact and rewrite weak bullets into achievement bullets.
Then create:
1) 1-page concise version
2) 2-page full version
3) a short professional summary
4) a cover letter master template
Keep the final output in Thai first, then provide the English resume copy.
```

### Prompt: Portfolio README

```text
I will give you a GitHub repository and rough project notes.
Write a senior-level README that helps remote employers understand value fast.
Include:
- what problem it solves
- architecture summary
- key technical highlights
- why this is hard / interesting
- setup steps
- future roadmap
- 5 resume-ready bullet points
Keep it concise and credible.
```

## ข้อควรระวัง

อย่าใช้ Computer กับงานเล็กที่ตอบได้ด้วย Ask หรือ Research เพราะเครดิตถูกใช้กับฟีเจอร์ compute สูง และอาจไม่คุ้มถ้างานนั้นไม่ต้องเข้าเว็บจริง ([How Credits Work](https://www.perplexity.ai/help-center/en/articles/13838041-how-credits-work-on-perplexity)).

ข้อมูลจากชุมชนผู้ใช้สะท้อนว่าควรเก็บ task ใหญ่เป็นรอบ ๆ และบันทึก output สำคัญไว้ เพราะงานใหญ่บางประเภทอาจใช้ทรัพยากรสูงมาก ทั้งหมดนี้ควรมองเป็นประสบการณ์ผู้ใช้ ไม่ใช่นโยบายทางการ ([Reddit export thread tip](https://www.reddit.com/r/perplexity_ai/comments/1rcoeic/hitting_perplexitys_unlimited_limits_heres_a/), [Reddit budget complaint](https://www.reddit.com/r/perplexity_ai/comments/1rixe73/perplexity_max_computer_used_entire_monthly/)).
